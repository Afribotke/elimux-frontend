Docs/Overview

Skolex API

The Skolex API gives you programmatic access to 21,847 institutions across 186 countries. Search universities, TVETs, and polytechnics; query verified visa agents; and retrieve localised country configurations — all via a simple REST interface or MCP protocol.

🔍
Search API
Query institutions by subject, country, budget, qualification level and mode. Returns ranked results with AI match reasoning.
⚡
MCP Protocol
Use Skolex as a tool inside Claude, GPT, or any MCP-compatible AI agent. Ship education search in your AI product in minutes.
✈️
Agents API
Query the Skolex Verified Agent Directory. Filter by country, tier, and specialisation.
🧪
Playground
Test every endpoint live in the browser. See real JSON responses. No setup required.

Base URL

endpoint
https://api.skolex.io/v1

Quick Example

curl
curl -X POST https://api.skolex.io/v1/search \ -H "Authorization: Bearer sk_live_YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{ "query": "Medicine in Kenya under KES 500,000", "mode": "uni", "limit": 10, "country_context": "KE" }'

Authentication

All requests require a Bearer token in the Authorization header. Get your key from the API Keys section.

Versioning

The current version is v1. The version is included in the URL path. When breaking changes are introduced, a new version is released and the old version is supported for 12 months.

Quickstart

Get your first search result in under 5 minutes.

Step 1 — Get an API Key

Navigate to API Keys and create a new key. Copy it — it will only be shown once.

⚠️Never expose your API key in client-side JavaScript. Always call the Skolex API from your backend server.

Step 2 — Make your first search

JavaScript
Python
cURL
javascript
const response = await fetch('https://api.skolex.io/v1/search', { method: 'POST', headers: { 'Authorization': `Bearer ${process.env.SKOLEX_API_KEY}`, 'Content-Type': 'application/json', }, body: JSON.stringify({ query: 'Medicine in Kenya under KES 500,000', mode: 'uni', limit: 10, country_context: 'KE', }) }); const { results } = await response.json(); results.forEach(r => console.log(r.institution, r.course, r.fees));

Docs/Authentication

Authentication

Skolex uses Bearer tokens for API authentication. Include your API key in every request.

API Key Types

sk_live_...
Live Key
For production use. Consumes your monthly quota. Rate limits apply.
sk_test_...
Test Key
For development and testing. Returns mock data. Does not consume quota.

Request Format

http header
Authorization: Bearer sk_live_YOUR_API_KEY_HERE
🚨
Never expose your key client-side. If a key is compromised, rotate it immediately from the API Keys dashboard. Compromised keys are your responsibility.

Securing Your Key

.env file
# .env — never commit this file SKOLEX_API_KEY=sk_live_REPLACE_WITH_YOUR_KEY SKOLEX_ENV=production
node.js backend
// ✅ Correct — key only on server app.post('/api/search', async (req, res) => { const r = await fetch('https://api.skolex.io/v1/search', { headers: { 'Authorization': `Bearer ${process.env.SKOLEX_API_KEY}` }, // ... }); res.json(await r.json()); }); // ❌ WRONG — never do this // fetch('https://api.skolex.io/v1/search', { // headers: { 'Authorization': 'Bearer sk_live_abc123' } // exposed! // })

Docs/REST API/Search

Search Endpoint

The core Skolex search — returns ranked institution and programme recommendations powered by AI. Supports university, TVET, and polytechnic modes.

POST /v1/search AI-powered course & institution search stable

Request Body

ParameterTypeRequiredDescription
querystringrequiredNatural language search query. e.g. "Medicine in Kenya under KES 500,000"
modestringoptionaluni (default) or tvet. Switches between university and vocational modes.
limitintegeroptionalNumber of results. Default: 10. Max: 10.
country_contextstringoptionalISO country code (e.g. KE, NG, GB). Localises qualification names, currency, and TVET authority labels.
kcse_gradestringoptionalStudent's entry grade (e.g. C-). Used in TVET mode to filter qualifying programmes.
languagestringoptionalen (default), sw (Kiswahili), fr (French). Affects response language.

Response Fields

FieldTypeDescription
rankintegerPosition 1–10
institutionstringFull institution name
countrystringCountry name
flagstringCountry flag emoji
coursestringSpecific programme name
levelstringBachelors, Masters, PhD, Diploma, Certificate, Short Course
feesstringAnnual fees in local currency
fees_usdstringApproximate USD equivalent
durationstringProgramme duration e.g. "4 years", "12 months"
modestringOn campus, Online, or Hybrid
scholarshipbooleanWhether scholarships are available
match_reasonstringAI explanation of why this result matches the query
apply_urlstringDirect application link on Skolex
tveta_approvedbooleanTVET mode only. Whether the programme is accredited by the national TVET authority.
min_kcsestringTVET mode only. Minimum entry qualification (e.g. "D+" or "Open entry")
pathwayarray<string>TVET mode only. Degree progression routes from this programme.
200 OK 400 Bad Request — missing query 401 Unauthorized — invalid key 429 Too Many Requests — rate limit

Errors & Status Codes

Skolex uses standard HTTP status codes. All errors return a JSON body with a machine-readable code and human-readable message.

error format
{ "error": { "code": "invalid_api_key", "message": "The API key provided is invalid or has been revoked.", "status": 401 } }

Docs/MCP/Overview

MCP Protocol

Skolex implements the Model Context Protocol (MCP), allowing any MCP-compatible AI agent — Claude, GPT, custom LLMs — to call Skolex search natively as a tool, without any extra integration work.

✓MCP v1.1.0 is live — the schema is already embedded in the Skolex Ask platform. Point your MCP client at https://api.skolex.io/mcp to connect.

What MCP enables

With MCP, instead of writing custom code to call the Skolex REST API, you can simply give Claude (or any MCP agent) access to the Skolex tool and say "Find me 10 medical schools in Kenya" — Claude will call Skolex internally and return structured results.

MCP Endpoint

mcp
Transport: HTTP (GET/POST) Endpoint: https://api.skolex.io/mcp Version: 1.1.0 Tools: skolex_search · skolex_agents · skolex_countries

MCP Schema (live)

json
{ "name": "skolex", "version": "1.1.0", "description": "Global education discovery — universities, TVETs, and verified visa agents", "tools": [ { "name": "skolex_search", "description": "Search 21,847 institutions across 186 countries", "input_schema": { "type": "object", "properties": { "query": { "type": "string", "description": "Natural language search" }, "mode": { "type": "string", "enum": ["uni", "tvet"], "default": "uni" }, "limit": { "type": "integer", "default": 10 }, "country_context": { "type": "string", "description": "ISO country code" }, "kcse_grade": { "type": "string", "description": "Entry grade for TVET mode" } }, "required": ["query"] } }, { "name": "skolex_agents", "description": "Get verified visa agents from Skolex directory", "input_schema": { "type": "object", "properties": { "country": { "type": "string" }, "tier": { "type": "string", "enum": ["gov", "premium", "verified"] } } } } ] }

Use Skolex with Claude

Connect Skolex to Claude via the Anthropic API's MCP server support. Claude will call Skolex natively when asked education questions.

Via Anthropic API

javascript
const response = await fetch('https://api.anthropic.com/v1/messages', { method: 'POST', headers: { 'x-api-key': process.env.ANTHROPIC_API_KEY, 'anthropic-version': '2023-06-01', 'Content-Type': 'application/json', }, body: JSON.stringify({ model: 'claude-sonnet-4-20250514', max_tokens: 1024, messages: [{ role: 'user', content: 'Find 5 medical schools in Kenya under KES 400k/yr' }], // Attach Skolex as an MCP server mcp_servers: [{ type: 'url', url: 'https://api.skolex.io/mcp', name: 'skolex', }] }) });

In Claude.ai Desktop

Add Skolex to your Claude.ai MCP configuration file:

claude_desktop_config.json
{ "mcpServers": { "skolex": { "command": "npx", "args": ["-y", "@skolex/mcp-server"], "env": { "SKOLEX_API_KEY": "sk_live_YOUR_KEY" } } } }

JavaScript / Node.js SDK

Official Skolex JavaScript SDK — works in Node.js and modern browsers via your backend proxy.

⚠️SDK package @skolex/sdk will be published to npm once the backend is deployed. The interface below is final — you can build against it now.

Installation

terminal
npm install @skolex/sdk

Initialise

javascript
import Skolex from '@skolex/sdk'; const skolex = new Skolex({ apiKey: process.env.SKOLEX_API_KEY, country: 'KE', // default country context timeout: 15000, // ms });

University Search

javascript
const results = await skolex.search.universities({ query: 'Medicine in Kenya under KES 500,000', limit: 10, }); results.forEach(r => { console.log(`${r.rank}. ${r.institution} — ${r.course}`); console.log(` ${r.fees} / ${r.duration} / ${r.mode}`); });

TVET Search

javascript
const results = await skolex.search.tvet({ query: 'Electrical installation Nairobi', grade: 'D+', // KCSE grade filter limit: 10, }); results.forEach(r => { console.log(r.institution, r.course); console.log(` Min: ${r.min_kcse} · ${r.tveta_approved ? '✓ TVETA' : ''}`); if (r.pathway.length) console.log(` Pathway: ${r.pathway[0]}`); });

Error Handling

javascript
try { const results = await skolex.search.universities({ query: '...' }); } catch (error) { if (error instanceof Skolex.RateLimitError) { // Retry after error.retryAfter seconds await sleep(error.retryAfter * 1000); } else if (error instanceof Skolex.AuthError) { console.error('Invalid API key'); } else { throw error; } }

Handling Rate Limits

javascript
async function searchWithRetry(query, retries = 3) { for (let i = 0; i < retries; i++) { const r = await fetch('https://api.skolex.io/v1/search', { /* ... */ }); if (r.status === 429) { const wait = parseInt(r.headers.get('Retry-After')) || 60; await new Promise(resolve => setTimeout(resolve, wait * 1000)); continue; } return r.json(); } throw new Error('Rate limit retries exhausted'); }

Docs/Webhooks

Webhooks

Webhooks let Skolex notify your server when events happen — ad impressions, new scholarship applications, payment confirmations.

ℹ️Webhooks are available on Starter plan and above. Webhook delivery requires a publicly accessible HTTPS endpoint.

Event Types

EventTrigger
search.completedA student completes a search (opt-in, anonymised)
ad.impressionYour ad is shown to a student
ad.clickStudent clicks your ad
scholarship.enquiryStudent enquires about your scholarship
payment.succeededSubscription payment confirmed
payment.failedSubscription payment failed
agent.verifiedAgent application approved

Webhook Payload

json
{ "id": "evt_01H8K...", "type": "ad.click", "created": 1719878400, "data": { "ad_id": "uon-featured", "category": "edu", "query_context": "Medicine Kenya budget", "country": "KE", "timestamp": "2026-07-01T10:32:11Z" } }

Verify webhook signatures

node.js
import crypto from 'crypto'; function verifyWebhook(payload, signature, secret) { const expected = crypto .createHmac('sha256', secret) .update(payload) .digest('hex'); return crypto.timingSafeEqual( Buffer.from(expected), Buffer.from(signature) ); }
