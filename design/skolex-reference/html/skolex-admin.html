🔐 Security Checklist
✓
Input sanitisation (escHtml)
All user inputs are escaped before rendering into the DOM. XSS injection blocked at render level.
Done
✓
API key not exposed in client HTML
The Anthropic API key is not hardcoded in the frontend. Currently called via browser fetch — needs backend proxy.
Done
✕
Backend API proxy required
Claude API calls go directly from browser to Anthropic. Must route through your own backend (/api/v1/search) to protect the API key and enforce rate limits.
✕
Rate limiting — not implemented
No limit on how many searches a user can make. Risk of API cost abuse. Implement: 10 searches/IP/hour client-side + 50/hour server-side.
✕
Content Security Policy (CSP) missing
No CSP headers. Malicious scripts could run from injected ad content. Add CSP meta tag: default-src 'self'; script-src 'self'.
✕
Admin authentication is client-side only
Current admin login is JavaScript-only with hardcoded credentials. Anyone who reads the source code can bypass it. Needs a real backend auth layer (JWT + bcrypt).
✕
GDPR / Data consent notice missing
You are collecting search queries, country preferences and email addresses. A consent notice and privacy acknowledgement is legally required, especially for UK/EU users.
✕
Advertiser form has no spam protection
The advertise and agent application forms have no honeypot, CAPTCHA or bot detection. Will receive bot submissions immediately on launch.
!
Agent licence verification is manual only
Agent licences are not automatically verified against any registry. Admin must manually check each licence number. Consider an automated verification API.
!
HTTPS enforcement
Must be enforced at the web server/CDN level. Ensure all traffic redirects from HTTP to HTTPS. Add HSTS header.
